## 한 문장 요약

**상용화를 포기한 게 아니라, 상용화의 족쇄를 벗어던진 것.**

## 1. 핵심 철학: 5원칙

### 원칙 1 — 지저분한 영역을 점령한다

오버레이, 실시간 방송 효과, 음성 파이프라인, 화면 녹화 도구.
이런 앱은 지저분하고, 돈이 안 되고, 유지보수가 더럽다.
**그래서 아무도 안 한다. 그래서 우리가 한다.**

경쟁이 없는 영역에서 자기만의 도구를 갖는 건 비대칭 우위다.

### 원칙 2 — Play Store에 가지 않는다

Play Store에 올리는 순간 발생하는 것들:
- targetSdk 매년 강제 업그레이드
- 정책 변경 대응 (권한, 백그라운드, 광고)
- 불특정 다수 기기 호환성 테스트
- 사용자 리뷰 관리
- **무료 앱이면 이 모든 게 순수 비용**

안 올리면 이 모든 게 **제로**다.
삼성 하드코딩, 권한 전체 허용, minSdk 고정 — 상용 앱에선 불가능한 자유가 전부 가능해진다.

### 원칙 3 — 코드를 한 줄도 치지 않는다

모든 개발은 Claude Code에게 **말로** 시킨다.
바로 안 되는 거 안다. 리니지 마이닝하듯이 레벨업한다.
Parksy Pen v1.0.31 — 31번 빌드한 이력이 증거다.

족발 썰면서 일하고, 퇴근할 때 하나 얻어가는 개발.
부담 제로, 압박 제로.

### 원칙 4 — On-Device, 과금 제로

삼성 폰에 커스터마이징한다. 클라우드 API에 의존하지 않는다.
- whisper.cpp 온디바이스 → STT 과금 제로
- 로컬 FFmpeg → 오디오 처리 과금 제로
- 삼성 녹음 폴더 직접 접근 → 클라우드 동기화 불필요

100% 정확도가 필요 없다. 개인 도구에 80%면 충분하다.
모델은 좋아지고, 폰은 빨라진다. 시간이 이 문제를 푼다.

### 원칙 5 — 최소 GUI, 진짜 필요한 수작업만

사용자가 1명이다. 불편하면 "이거 고쳐"라고 말하면 끝이다.
버튼 하나, 토글 하나가 최선이다.
GUI가 복잡해지면 유지보수가 기하급수적으로 는다.
앱 10개를 혼자 운용하려면 이건 생존 전략이다.

## 2. APK는 엔드 프로덕트가 아니다

APK는 **팹 라인의 장비**다.

### 콘텐츠 생태계 플라이휠

```
APK 개발 과정 ──→ YouTube 시리즈 (개발기)
     │                @technician-parksy
     ▼
APK 도구 사용 ──→ YouTube 콘텐츠 (활용기)
     │                도구로 만든 영상을 또 업로드
     ▼
parksy.kr ──→ 페르소나 "박씨" 방송국
     │                콘텐츠 허브
     ▼
eae.kr ──→ PatchTech 교육 시리즈
     │        AI 시대 물성화 스킬셋 6가지 중 하나
     ▼
IoT 하드웨어 ──→ 중국 칩 + Claude Code
                   커스텀 디바이스 제작 방송
```

### 하나를 만들면 5군데에서 써먹는다

APK 하나가 동시에:

| 역할 | 행선지 |
|------|--------|
| 내가 쓰는 **도구** | 실전 방송/작업 |
| 만드는 과정이 **방송 소재** | YouTube 개발기 시리즈 |
| 도구로 만든 결과물이 **방송 소재** | YouTube 활용기 시리즈 |
| 스토어 링크가 **시청자 접점** | dtslib-apk-lab.vercel.app |
| 교육 플랫폼 **커리큘럼** | eae.kr PatchTech |

## 4. 장기 로드맵

### Phase: 소프트웨어 (현재)
- 방송 도구 APK 무기고 완성
- 후편집 없이 화면 녹화만으로 방송 효과 구현
- On-Device 전환 (API 과금 제거)

### Phase: 하드웨어 (다음)
- 저가 중국 IoT 제품 + 커스텀 칩
- Claude Code로 펌웨어 개발
- 직접 움직이는 장난감, 필요한 제품 패치
- 제작 과정 자체가 방송 콘텐츠

### Phase: 콘텐츠 팩토리 (목표)
- 팹 라인 가동 → 콘텐츠 양산
- YouTube, parksy.kr, eae.kr 동시 송출
- 도구 → 콘텐츠 → 교육 → 도구 개선 순환 루프

## 5. 이 문서의 의미

이 문서는 기술 문서가 아니다.
왜 이 레포지토리가 존재하는지, 왜 이렇게 만드는지를 정의하는 **헌장**이다.

기능 명세는 변한다. 코드는 리팩토링된다. 앱은 추가되고 삭제된다.
하지만 이 5원칙은 변하지 않는다.

> **지저분한 걸 만든다. 팔지 않는다. 말로 시킨다. 폰에서 돌린다. 심플하게 만든다.**
> **그리고 그 전부가 콘텐츠가 된다.**
