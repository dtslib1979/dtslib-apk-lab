## EP.01 — OBS급 방송 믹서 (파일럿)

### 제목 후보
- "갤럭시로 라방 세팅하는 법 (OBS 없이)"
- "폰 하나로 유튜브 라이브 방송국 만들기"
- "OBS가 Android에 없는 이유와 대안"

### 시연 시나리오

```
[인트로 2분]
PC에서 OBS 켜는 장면 → "이게 Android에 없음"
→ 유튜버 지망생의 고통 공감

[문제 정의 3분]
OBS가 Android에서 안 되는 진짜 이유:
- 다중 소스 캡처 (카메라 + 화면 + 마이크 동시)
- RTMP 스트리밍 + 씬 전환
- Android = 단일 화면 캡처 수준

[해결 과정 10분]
음성 입력 → Claude Code → Parksy Studio 파이프라인
- 태블릿 = 입력 단말기
- PC (WSL2) = 두뇌
- 실시간으로 방송 믹서 구현

[결과 시연 5분]
구현된 방송 파이프라인 실제 동작:
- 다중 소스 → 영상 믹싱
- YouTube Live RTMP 송출
- 씬 전환

[아웃트로 2분]
"이게 16만원으로 가능합니다"
→ 학원비 배율 언급
→ 다음 에피소드 예고 (영상편집)
```

### 필요 재료 (BOM)

| 재료 | 상태 | 비고 |
|------|------|------|
| Parksy Studio v2.0 코드 | ✅ 완료 | Axis-shell 패턴 |
| Parksy Glot (자막 오버레이) | ✅ v2.0 코드 완료 | 빌드 필요 |
| 갤럭시 태블릿 | ✅ 보유 | 입력 단말기 |
| WSL2 PC | ✅ 세팅 완료 | 두뇌 |
| RTMP 스트리밍 설정 | ⚠️ 미완 | YouTube Live 키 필요 |
| 화면 녹화 세팅 | ⚠️ 미완 | 방송용 화면 캡처 |

## EP.02 — 전문 영상편집 (DaVinci급)

### 핵심 포인트
- DaVinci Resolve = PC 전용, Android 버전 없음
- 폰 + 음성 + Claude Code로 멀티트랙 편집 파이프라인 구현
- FFmpeg + WebCodecs + WebAssembly 조합

### BOM
- Parksy Studio 트리머 기능 (v1.0 50% 구현)
- FFmpeg WASM → Android MediaCodec 전환 (v2.0 과제)
- 대안: WSL2에서 FFmpeg 처리 후 결과 전송

## EP.03 — DAW (음악 제작)

### 핵심 포인트
- Ableton/FL Studio = ASIO + VSTi 생태계 = Android 근본 불가
- 박씨 REAPER 세팅 이미 완료 (DAW 세션에서 진행 중)
- 방송 포맷: 폰에서 음성으로 "이런 느낌의 음악" → PC REAPER + VSTi → 완성

### 주의
- DAW 세션 별도 진행 중 → 이 세션에서 건들지 않음
